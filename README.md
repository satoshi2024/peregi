全画面ダイアログをネストすることは、ユーザー体験とシステム保守性の観点から避けるべきです。ユーザー側では、ネストされた画面を閉じるときに何度も操作を繰り返す必要が生じ、ナビゲーションが複雑化し、作業フローが中断されることで認知負荷が大幅に増加します。これは、タスク完了までのステップが増え、離脱の原因となります。開発側では、多層にわたるコンポーネント間のステート管理とデータの同期が複雑化し、Z-Indexやフォーカス管理のバグが発生しやすくなります。結果として、システムの保守性（メンテナンス性）が著しく低下し、開発コストの増加につながるため、代わりに単一の全画面ダイアログ内でステップ形式やタブ形式に分割するなどの設計を推奨します。
