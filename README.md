ダイアログ内にさらにダイアログを表示する構成は、一見すると機能を分離できるように見えるが、実際には操作経路が複雑化し、イベント制御や状態管理も煩雑になるため、保守性が低下する傾向がある。
ユーザー視点では、画面階層が深くなることで操作負荷が増し、どのダイアログで操作しているのか分かりにくくなる問題がある。
開発者視点では、親子間のパラメータ受け渡しや閉鎖イベントの制御が複雑になり、テストや改修時の影響範囲も広がる。
そのため、結果表示モジュールは検索モジュールと同一ダイアログ内に統合し、タブ切り替えや領域分割などで機能を整理する方針とした。これにより、UI構造の簡素化、イベント管理の一元化、保守性の向上を図る。
